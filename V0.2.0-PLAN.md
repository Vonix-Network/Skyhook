# Skyhook v0.2.0 — production-ready release plan

**Goal:** ship v0.2.0 as a 2-shot release. Everything below must work like a paid product before tagging.

## Scope (in)

### Session management (core ask)
- **Connection state machine**: `Disconnected → Connecting → Connected → Degraded → Closed`
- **Per-connection session pool**: at most ONE live SFTP session per connection, ever
- **Click-to-focus tabs**: clicking an open connection focuses its tab; never spawns a 2nd
- **Auto-reconnect with backoff** (3 attempts, 1s→2s→5s) on transient channel death
- **Heartbeat / liveness ping** every 30s (a no-op `stat .`) — flips tab to Degraded if it fails
- **Graceful disconnect on app quit / tab close**: send SSH disconnect properly
- **Concurrent op queue per session**: ops serialize through a Tokio mpsc channel so multiple frontend calls don't race the SFTP channel
- **Reconnect button** on Degraded/Closed banners (already in v0.1.1, polish UX)

### File operations
- **Recursive upload (folders)** with per-file progress
- **Recursive download (folders)** with per-file progress
- **Real progress bars** in the transfer queue (bytes streamed, not just done/not-done)
- **Pause / Resume / Cancel** on transfers
- **Drag-drop upload** from OS into the file list
- **Right-click context menu**: Open, Edit, Download, Rename, Delete, Copy path, Properties
- **Inline rename** (F2 / double-slow-click)
- **Multi-select with Shift+click and Ctrl+click**
- **Keyboard nav**: arrows, Enter to open, Backspace to go up, Del to delete, F5 refresh, Ctrl+L focus path bar

### Editor
- **Monaco editor tab-in-tab**: double-click a text file → opens editor view
- Syntax highlighting auto-detected by extension
- **Save (Ctrl+S)** writes back via SFTP, with dirty indicator
- **Large file guard**: refuse to open files > 10 MB in the editor

### Quality / production polish
- **Trust-on-first-use known_hosts** with fingerprint warning
- **Last-used connection** auto-focus on startup
- **Settings panel**: theme (system/dark/light), confirm-on-delete toggle, default editor wrap, transfer concurrency
- **Hidden file toggle** (Ctrl+H)
- **Sortable columns**: Name / Size / Modified / Perms
- **Recursive search** of the current remote directory (server-side via SFTP walk)
- **Loading skeletons** instead of "Loading…" text
- **Error toast system** (no more `alert()`)
- **About dialog** with version + commit SHA + license
- **First-run welcome** with "Create your first connection" CTA
- **Empty/error states** designed, not text
- **Window state persistence** (size, position, sidebar collapse)

### Build / CI hardening
- **Test suite**: unit tests for path normalization, vault encrypt/decrypt, connection state machine
- **Headless integration test**: spin up an in-process OpenSSH server in Rust tests, run actual SFTP ops against it (`russh::server` + tempdir)
- **CHANGELOG** updated per the standing rule
- **Code-signed Windows installer** is out of scope (needs a cert); document as future

## Architecture changes from v0.1

### Backend rewrite of `sftp.rs` into a proper subsystem

Replace the current `Session { sftp: Mutex<SftpSession>, ... }` lock-and-call pattern with a real actor:

```
SessionManager (one global)
  └─ SessionActor (one per active connection)
     ├─ SSH transport (russh::client::Handle)
     ├─ SFTP channel (russh_sftp::SftpSession)
     ├─ Op queue: mpsc<SessionOp>
     ├─ Heartbeat task: every 30s, stat ".", on fail → Degraded
     ├─ State: Arc<Mutex<SessionState>>
     └─ Auto-reconnect task: on Degraded, try N times with backoff
```

`SessionOp` enum encodes every operation (ListDir, ReadFile, etc.) with a `oneshot::Sender` for the reply. This kills:
- Race conditions on the SFTP channel
- Lock-ordering bugs
- Zombie sessions (state is explicit, not implicit "did the last call work")

### Transfer engine

```
TransferEngine (one global)
  ├─ Pending queue: VecDeque<TransferJob>
  ├─ Active set: 2 concurrent by default (configurable)
  ├─ Per-job: TransferState (Queued | Active | Paused | Done | Cancelled | Errored)
  └─ Progress channel: tauri::Event emits {id, bytes, total} every 100ms
```

Frontend subscribes to `transfer-progress` events instead of polling. Pause/Resume just stops/resumes copying.

## Waves (3-shot dispatch budget)

### Wave 1: backend foundation (parallel x3)
1. **Subagent A**: rewrite `sftp.rs` into actor pattern. New file `src-tauri/src/session/mod.rs` + `session/actor.rs` + `session/state.rs`. Delete old `Session` impl. All commands.rs callers updated to new API.
2. **Subagent B**: transfer engine. New `src-tauri/src/transfers/mod.rs`. Recursive folder walk for upload/download. Progress events.
3. **Subagent C**: known_hosts TOFU + settings persistence. New `src-tauri/src/settings.rs` (JSON file in config dir). New `src-tauri/src/known_hosts.rs` (line-based hosts file).

After Wave 1: main thread runs `cargo check`, fixes any integration drift, commits.

### Wave 2: frontend (parallel x3)
1. **Subagent D**: rewrite store + add toast system + settings panel + about dialog. New components/Settings, components/Toast, components/About.
2. **Subagent E**: Monaco editor tab. New components/Editor.tsx, mode switcher in tab UI.
3. **Subagent F**: file list polish — right-click menu, drag-drop, multi-select, keyboard nav, inline rename, sortable columns. Edits Browser.tsx.

After Wave 2: main thread integrates, fixes types, runs `pnpm build`, commits.

### Wave 3: tests + docs + release (main thread)
1. Integration test with russh server.
2. CHANGELOG entry for 0.2.0.
3. Bump versions to 0.2.0.
4. Build, tag, release.

## Schema contracts (pre-defined to avoid pitfall #4)

### Tauri command surface (Wave 1 deliverable, Wave 2 consumer)

```typescript
// Connection / Session
list_connections() -> Connection[]
save_connection(connection: Connection) -> Connection
delete_connection(id: string) -> void
connect(connectionId: string) -> SessionInfo
disconnect(sessionId: string) -> void
reconnect(sessionId: string) -> SessionInfo
session_status() -> SessionInfo[]

// File ops
list_dir(sessionId, path) -> DirEntry[]
read_file(sessionId, path) -> string
write_file(sessionId, path, content) -> void
make_dir(sessionId, path) -> void
remove_path(sessionId, path) -> void
rename(sessionId, from, to) -> void
stat(sessionId, path) -> DirEntry           // NEW
walk(sessionId, root) -> DirEntry[]         // NEW — recursive

// Transfers
transfer_enqueue(sessionId, jobs: TransferRequest[]) -> string[]   // returns ids
transfer_pause(id: string) -> void
transfer_resume(id: string) -> void
transfer_cancel(id: string) -> void
transfer_list() -> Transfer[]

// Settings
get_settings() -> Settings
save_settings(settings: Settings) -> void
```

### Event surface (Wave 1 emitter, Wave 2 listener)

```typescript
"session-state-changed": { sessionId: string, state: "connecting"|"connected"|"degraded"|"closed", reason?: string }
"transfer-progress":     { id: string, bytes: number, total: number | null, status: TransferStatus }
"session-heartbeat":     { sessionId: string, ok: boolean }
```

### Types (shared)

```typescript
type SessionState = "connecting" | "connected" | "degraded" | "closed"

interface SessionInfo {
  id: string
  connection_id: string
  state: SessionState
  cwd: string
  last_error: string | null
  connected_at: number
}

interface TransferRequest {
  direction: "upload" | "download"
  local: string
  remote: string
  recursive: boolean
}

interface Transfer {
  id: string
  session_id: string
  direction: "upload" | "download"
  local: string
  remote: string
  bytes: number
  total: number | null
  status: "queued" | "active" | "paused" | "done" | "cancelled" | "error"
  error: string | null
  started_at: number
}

interface Settings {
  theme: "system" | "dark" | "light"
  confirm_on_delete: boolean
  editor_word_wrap: boolean
  transfer_concurrency: number
  last_active_connection_id: string | null
  window: { width: number; height: number; x: number | null; y: number | null }
}
```
